🪟 Windows 安装指南（WSL2）

本教程专为 Windows 用户设计，使用 WSL2（Windows Subsystem for Linux）在 Windows 上运行龙虾 AI。不需要懂 Linux，跟着步骤做即可。

💡 为什么用 WSL2？
  • 在 Windows 上获得完整的 Linux 环境
  • 和 macOS/Linux 用户使用体验一致
  • 避免 Windows 特有的各种兼容性问题
  • 微软官方支持，稳定可靠
⚠️ 系统要求：
  • Windows 10 版本 2004 或更高（或 Windows 11）
  • 至少 8GB 内存（推荐 16GB）
  • 至少 20GB 可用磁盘空间
  • 需要管理员权限

📋 安装步骤总览

  • 启用 WSL2 功能
  • 安装 Ubuntu
  • 安装 Git、Node.js、pnpm
  • 克隆项目代码
  • 安装依赖并配置
  • 启动服务

第 1 步：启用 WSL2 功能

1.1 以管理员身份打开 PowerShell

1 按 Win + X，选择 "Windows PowerShell (管理员)" 或 "终端 (管理员)"。
📝 提示：如果看到用户账户控制提示，点击"是"。

1.2 运行 WSL 安装命令

2 在 PowerShell 中输入以下命令，按回车：
wsl --install
3 等待安装完成。这个过程可能需要几分钟，会自动：
  • 启用 WSL 功能
  • 启用虚拟机平台功能
  • 下载并安装 Ubuntu（默认版本）

1.3 重启电脑

4 安装完成后，必须重启电脑才能生效。
⚠️ 重要：不要跳过重启！很多用户没重启导致后续步骤失败。

第 2 步：设置 Ubuntu 用户

2.1 首次启动 Ubuntu

1 重启后，系统会自动打开 Ubuntu 窗口。如果没有，按 Win 键，搜索 "Ubuntu" 并打开。

2.2 创建用户名和密码

2 按提示输入用户名（建议用英文，比如 siwei），按回车。
3 输入密码。注意：输入密码时屏幕上不会显示任何字符，这是正常的，输完按回车即可。
4 再次输入密码确认，按回车。
✅ 完成！现在你已经进入 Ubuntu 系统了。你会看到类似 username@DESKTOP-XXXX:~$ 的提示符。

第 3 步：安装必要软件

3.1 更新软件包列表

1 在 Ubuntu 窗口中，输入以下命令：
sudo apt update && sudo apt upgrade -y
📝 提示：输入密码时不会显示字符，输完按回车。这个过程可能需要几分钟。

3.2 安装 Git

sudo apt install -y git
2 验证安装：
git --version
✅ 成功标志：显示类似 git version 2.x.x

3.3 安装 Node.js（使用 nvm）

3 安装 nvm（Node 版本管理器）：
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
4 重新加载配置：
source ~/.bashrc
5 安装 Node.js 20（推荐版本）：
nvm install 20
6 验证安装：
node --version && npm --version
✅ 成功标志：显示 v20.x.x 和 npm 版本号

3.4 安装 pnpm

npm install -g pnpm
7 验证安装：
pnpm --version

第 4 步：克隆项目代码

4.1 选择代码存放位置

1 在你的用户目录下创建项目文件夹：
cd ~ && mkdir -p projects && cd projects

4.2 克隆代码

2 输入克隆命令（替换成实际的项目地址）：
git clone https://github.com/你的项目地址.git
📝 提示：如果是私有仓库，需要配置 SSH 或使用带 token 的 HTTPS 地址。

4.3 进入项目目录

cd 项目文件夹名

第 5 步：安装依赖并配置

5.1 安装项目依赖

pnpm install
📝 提示：第一次安装可能需要几分钟，等待完成。

5.2 配置 API Key

1 复制环境变量示例文件：
cp .env.example .env
2 编辑配置文件：
nano .env
📝 提示：在 nano 编辑器中，用方向键移动，修改后按 Ctrl+O 保存，Ctrl+X 退出。
3 填入你的 API Key（参考 API Key 教程）：
DASHSCOPE_API_KEY=sk-你的 key

第 6 步：启动服务

6.1 启动开发服务器

pnpm dev
✅ 成功标志：看到类似 Server running at http://localhost:5173 的提示

6.2 在浏览器中访问

1 打开 Windows 的浏览器（Chrome/Edge 等）
2 访问 http://localhost:5173
📝 提示：WSL2 中运行的服务可以直接通过 localhost 在 Windows 浏览器中访问。

🔧 常见问题排查

❌ "wsl: 无法将 wsl 项识别为 cmdlet"

原因：WSL 功能未启用或系统版本太旧

解决：

  1. 确认 Windows 版本是 2004 或更高（设置 → 系统 → 关于）
  2. 以管理员身份运行 PowerShell，执行：wsl --install
  3. 重启电脑

❌ "无法访问网络" 或 "下载失败"

原因：WSL2 网络配置问题

解决：

  1. 在 Ubuntu 中执行：sudo service networking restart
  2. 或者重启 WSL：wsl --shutdown（在 PowerShell 中执行），然后重新打开 Ubuntu

❌ "node: command not found"

原因：nvm 未正确加载

解决：

  1. 执行：source ~/.bashrc
  2. 再执行：node --version 验证

❌ "权限不足" 或 "Permission denied"

原因：文件权限问题

解决：

  1. 在项目目录执行：sudo chown -R $USER:$USER .
  2. 或者用 sudo 运行命令（不推荐长期使用）

❌ 端口被占用

原因：5173 端口已被其他程序使用

解决：

  1. 换一个端口启动，比如：pnpm dev -- --port 5174
  2. 或者关闭占用端口的程序
